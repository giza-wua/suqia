## نظرة عامة

**سُقيا** منصة ويب متكاملة لإدارة وتوثيق الأصول الهندسية لري محافظة الجيزة، مبنية بالكامل باستخدام Vanilla JS (من دون أُطر عمل — frameworks) وقاعدة بيانات Firebase Firestore، وتدعم الهاتف المحمول والحاسوب بالجودة نفسها.

| الوحدة | الوصف | مجموعة Firestore |
|--------|-------|----------------------|
| 📊 لوحة التحكم | نظرة عامة، وبحث موحّد في جميع البيانات | — |
| 🌊 ترع الإدارة | الأورنيك الهندسي للترع (المقاطع، والمناوبات، والتبطين) | `specs` + `linedCanals` |
| 🌉 الكباري | كباري الترع والمصارف | `bridges` |
| 🚰 المصارف | مصارف الصرف الزراعي (التصريف بالجاذبية، من دون نظام مناوبة) | `drains` |
| 💧 الآبار الجوفية | آبار المياه الجوفية وبيانات الطلمبات | `wells` |
| 🗺️ الخريطة | خريطة تفاعلية موحّدة لجميع الأصول ذات الإحداثيات الجغرافية | — |
| ⚙️ الإعدادات | الحساب، والمستخدمون، وسجل النشاط، وقاعدة البيانات | `users`, `activityLog` |

## الميزات الرئيسية

- 🔐 **مصادقة حقيقية** عبر Firebase Authentication (اسم مستخدم وكلمة مرور عاديان، من دون أن يظهر أي بريد إلكتروني للمستخدم)
- 👥 **ثلاثة مستويات للصلاحيات**: 👑 مسؤول (صلاحية كاملة) / ✏️ محرر (إضافة وتعديل البيانات) / 👁️ مشاهد (عرض فقط)
- 📋 **سجل نشاط كامل** — تسجيل الدخول والخروج، إضافة إلى كل عملية إضافة أو تعديل أو حذف على البيانات، ويظهر للمسؤول من تبويب الإعدادات
- 📱 **مصمَّم على مبدأ mobile-first بالكامل**: شريط تنقل سفلي، وزر إضافة عائم (FAB)، وجميع الصفحات مُختبَرة فعلياً على الشاشات الضيقة
- 📲 **قابل للتثبيت (PWA)** — يُثبَّت على الشاشة الرئيسية للهاتف المحمول، ويعمل حتى من دون اتصال بالإنترنت
- 📄 **تصدير إلى Excel وPDF** يلتزم بالفلاتر النشطة ويوضّحها في عنوان الملف، مع تقرير PDF يحمل شعار الأمانة العامة الرسمي وبلوك اعتماد في نهايته
- 🌐 **دعم العمل دون اتصال للقراءة** — تظهر آخر نسخة محفوظة تلقائياً في حال انقطاع الاتصال بالإنترنت
- 🔍 **بحث موحّد** في لوحة التحكم يبحث في جميع الأصول في آن واحد
- 🛠️ **تنبيهات صيانة** لأي سجل لم يُحدَّث منذ أكثر من سنة
- 📊 **رسوم بيانية تحليلية** في لوحة التحكم (توزيع الكباري والآبار)
- 🌙 وضعان ليلي ونهاري، وتصميم عربي من اليمين إلى اليسار (RTL) بخط Cairo محلي

## التشغيل السريع

### المتطلبات
- مشروع Firebase مُفعَّل عليه Firestore وAuthentication
- خادم ويب ثابت (VS Code Live Server، أو Python، أو Nginx، أو Cloudflare Pages)

### الخطوات

1. **ضبط إعدادات Firebase** — في الملف `js/firebase-config.js`، تُدرَج بيانات المشروع الخاص بك.
2. **تفعيل Firebase Authentication** — عبر موفِّر Email/Password (خطوة يدوية إلزامية تُنفَّذ مرة واحدة).
3. **نشر قواعد Firestore** — يُنسخ محتوى الملف `firestore.rules` ويُنشر من Firebase Console.
4. **تشغيل الخادم محلياً:**
   ```bash
   python -m http.server 7070
   # أو
   npx serve .
   ```
5. **تسجيل الدخول لأول مرة** بحساب المسؤول الأول — يُرحَّل تلقائياً إلى نظام المصادقة الجديد.
6. **استيراد البيانات:** من الإعدادات ← ابدأ الاستيراد.

الخطوتان 2 و3 **إلزاميتان**، وموضَّحتان بالتفصيل خطوة بخطوة في [`docs/SETUP.md`](docs/SETUP.md) — فمن دونهما لن يعمل تسجيل الدخول.

## دعم الهاتف المحمول

بُني التطبيق على مبدأ mobile-first بالكامل:
- شريط تنقل سفلي ثابت (Bottom Navigation) في جميع الصفحات
- زر إضافة عائم (FAB) بدلاً من وضعه داخل شريط العنوان المزدحم
- البحث الموحّد موجود في لوحة التحكم فقط، تفادياً لازدحام شريط العنوان في بقية الصفحات
- مساحات لمس مريحة لجميع الأزرار (40 بكسل فأكثر)
- جميع الصفحات جرى اختبارها فعلياً على عرض 375 بكسل من دون أي تمدد أفقي أو محتوى مبتور

## نظام المصادقة والصلاحيات

- **مصادقة حقيقية عبر Firebase Authentication** — اسم مستخدم وكلمة مرور عاديان، يتحوّلان تلقائياً إلى بريد إلكتروني داخلي وهمي (`username@suqia.local`) من دون أن يلحظ المستخدم أي فرق
- تتحقق قواعد Firestore (`firestore.rules`) فعلياً من الهوية والصلاحية عبر `request.auth`
- تُحفظ الجلسات في `sessionStorage` (للواجهة) بالإضافة إلى جلسة Firebase Auth حقيقية (للحماية الفعلية)
- تُرحَّل الحسابات القديمة تلقائياً وبصمت إلى Firebase Auth عند أول تسجيل دخول ناجح
- ثلاث صلاحيات: 👑 مسؤول / ✏️ محرر / 👁️ مشاهد — والمسؤول وحده يملك صلاحية إدارة المستخدمين

> ⚠️ **قيود معروفة (نظراً لعدم وجود خادم خلفي):** لا يستطيع المسؤول تغيير كلمة مرور مستخدم آخر مباشرة (يقوم كل مستخدم بتغييرها بنفسه)، ولا يمكن إنشاء مستخدم جديد بالاسم نفسه لمستخدم سبق حذفه. التفاصيل الكاملة وأسبابها التقنية وحلولها المستقبلية موضَّحة في [`docs/SETUP.md`](docs/SETUP.md).
