## نظرة عامة

**سُقيا** منصة ويب متكاملة لإدارة وتوثيق الأصول الهندسية لري محافظة الجيزة، مبنية بالكامل بـ Vanilla JS (بدون frameworks) وقاعدة بيانات Firebase Firestore، وتدعم الموبايل والكمبيوتر بنفس الجودة.

| الوحدة | الوصف | Firestore Collection |
|--------|-------|----------------------|
| 📊 لوحة التحكم | نظرة عامة + بحث موحّد على كل البيانات | — |
| 🌊 ترع الإدارة | الأورنيك الهندسي للترع (مقاطع + مناوبات + تبطين) | `specs` + `linedCanals` |
| 🌉 الكباري | كباري الترع والمصارف | `bridges` |
| 💧 الآبار الجوفية | آبار المياه الجوفية وبيانات الطلمبات | `wells` |
| 🗺️ الخريطة | خريطة تفاعلية موحّدة لكل الأصول اللي ليها إحداثيات | — |
| ⚙️ الإعدادات | الحساب، المستخدمون، سجل النشاط، قاعدة البيانات | `users`, `activityLog` |

## الميزات الرئيسية

- 🔐 **مصادقة حقيقية** عبر Firebase Authentication (اسم مستخدم وكلمة مرور عادي، بدون إيميل ظاهر للمستخدم)
- 👥 **3 مستويات صلاحيات**: 👑 مسؤول (كل شيء) / ✏️ محرر (إضافة وتعديل البيانات) / 👁️ مشاهد (عرض فقط)
- 📋 **سجل نشاط كامل** — دخول/خروج + كل إضافة/تعديل/حذف على البيانات، بيظهر للمسؤول من تبويب الإعدادات
- 📱 **مبني mobile-first بالكامل**: شريط تنقل سفلي، زرار إضافة عائم (FAB)، كل الصفحات مفحوصة فعلياً على شاشات ضيقة
- 📲 **قابل للتثبيت (PWA)** — يتثبّت على شاشة الموبايل الرئيسية ويفتح حتى بدون إنترنت
- 📄 **تصدير Excel وPDF** يلتزم بالفلاتر النشطة ويوضّحها في عنوان الملف — تقرير PDF بشعار الأمانة العامة الرسمي وبلوك اعتماد في النهاية
- 🌐 **دعم Offline للقراءة** — آخر نسخة محفوظة بتظهر تلقائياً لو النت انقطع
- 🔍 **بحث موحّد** في لوحة التحكم يدوّر في كل الأصول مع بعض
- 🛠️ **تنبيهات صيانة** لأي سجل ما اتحدّثش من أكتر من سنة
- 📊 **رسوم تحليلية** في لوحة التحكم (توزيع الكباري والآبار)
- 🌙 وضع ليلي/نهاري، تصميم عربي RTL بخط Cairo محلي

## التشغيل السريع

### المتطلبات
- مشروع Firebase مع Firestore + Authentication مُفعَّلين
- خادم ويب ثابت (VS Code Live Server / Python / Nginx / Cloudflare Pages)

### الخطوات

1. **ضبط Firebase** — في `js/firebase-config.js` ضع بيانات مشروعك.
2. **فعّل Firebase Authentication** — Email/Password provider (خطوة يدوية مرة واحدة، إجبارية).
3. **انشر قواعد Firestore** — انسخ محتوى `firestore.rules` وانشرها من Firebase Console.
4. **تشغيل الخادم محلياً:**
   ```bash
   python -m http.server 7070
   # أو
   npx serve .
   ```
5. **أول تسجيل دخول:** `maged` / (كلمة المرور المتفق عليها) — هيترحّل تلقائياً لنظام المصادقة الجديد.
6. **استيراد البيانات:** الإعدادات ← ابدأ الاستيراد

خطوات 2 و3 **إجبارية** وموضّحة بالتفصيل خطوة بخطوة في [`docs/SETUP.md`](docs/SETUP.md) — من غيرها تسجيل الدخول مش هيشتغل.

## دعم الموبايل

التطبيق مبني mobile-first بالكامل:
- شريط تنقل سفلي ثابت (Bottom Navigation) في كل الصفحات
- زرار الإضافة عائم (FAB) بدل ما يكون جوه شريط العنوان المزدحم
- البحث الموحّد موجود في لوحة التحكم فقط (عشان ميضغطش على شريط العنوان في باقي الصفحات)
- مساحات لمس مريحة لكل الأزرار (≥40px)
- كل الصفحات مفحوصة فعلياً على عرض 375px بدون أي تمدد أفقي أو محتوى مقصوص

## نظام المصادقة والصلاحيات

- **Firebase Authentication حقيقي** — اسم مستخدم وكلمة مرور عادي، بيتحوّل لإيميل داخلي وهمي (`username@suqia.local`) تلقائياً من غير ما المستخدم يحس بفرق
- قواعد Firestore (`firestore.rules`) بتتحقق فعلياً من الهوية والصلاحية عبر `request.auth`
- جلسات محفوظة في `sessionStorage` (للواجهة) + جلسة Firebase Auth حقيقية (للحماية الفعلية)
- حسابات قديمة بترحّل تلقائياً وبصمت لـ Firebase Auth أول مرة تسجّل دخول بنجاح
- 3 صلاحيات: 👑 مسؤول / ✏️ محرر / 👁️ مشاهد — المسؤول بس يقدر يدير المستخدمين

> ⚠️ **قيود معروفة (بدون خادم خلفي):** المسؤول مش يقدر يغيّر كلمة مرور مستخدم تاني مباشرة (كل مستخدم بيغيّرها بنفسه)، ومتقدرش تنشئ مستخدم جديد بنفس اسم مستخدم اتحذف قبل كده. التفاصيل الكاملة وأسبابها التقنية وحلولها المستقبلية في [`docs/SETUP.md`](docs/SETUP.md).
